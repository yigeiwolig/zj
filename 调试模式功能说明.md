# 调试模式功能说明

## 📍 概述
本文档列出了代码中所有**仅在开发工具环境（devtools）下才开启或跳过**的功能。

## 🔍 检测方式
代码中使用以下方式检测开发工具环境：
```javascript
const deviceInfo = wx.getDeviceInfo();
const isDevTools = deviceInfo.platform === 'devtools';
```

## 📋 调试模式下的特殊行为

### 1. **app.js** - 应用级别

#### 1.1 跳过封禁状态检查（onLaunch）
**位置**: `miniprogram/app.js` 第 306-317 行

**功能**: 应用启动时跳过封禁状态检查
```javascript
// 开发环境下跳过封禁检查，避免误判和自动解封
const isDevTools = deviceInfo.platform === 'devtools';
if (!isDevTools) {
  this.checkBanStatusOnLaunch();
} else {
  console.log('[app] 开发工具环境，跳过封禁状态检查');
}
```

**影响**: 
- ✅ 开发工具中不会触发封禁检查
- ✅ 可以正常调试，不会被封禁页面拦截

---

#### 1.2 跳过PC端检测
**位置**: `miniprogram/app.js` 第 333-337 行

**功能**: 跳过PC端（Windows/Mac微信）封禁检测
```javascript
// 🔴 开发工具环境下跳过检测，允许开发调试
if (platform === 'devtools') {
  console.log('[app] 开发工具环境，跳过PC端检测');
  return;
}
```

**影响**:
- ✅ 开发工具中不会触发PC端封禁
- ✅ 可以在开发工具中正常使用

---

#### 1.3 跳过封禁检查（checkBanStatusOnLaunch）
**位置**: `miniprogram/app.js` 第 384-390 行

**功能**: 应用启动时检查封禁状态，开发工具环境跳过
```javascript
// 🔴 开发环境下跳过封禁检查，避免误判
const isDevTools = deviceInfo.platform === 'devtools';
if (isDevTools) {
  console.log('[app] 开发工具环境，跳过封禁检查');
  return;
}
```

**影响**:
- ✅ 开发工具中不会检查封禁状态
- ✅ 可以正常调试所有功能

---

#### 1.4 跳过 qiangli 强制封禁检查
**位置**: `miniprogram/app.js` 第 506-511 行

**功能**: 定时检查 qiangli 强制封禁状态，开发工具环境跳过
```javascript
// 🔴 开发环境下跳过封禁检查，避免误判
const isDevTools = deviceInfo.platform === 'devtools';
if (isDevTools) {
  return; // 开发工具环境，直接返回
}
```

**影响**:
- ✅ 开发工具中不会触发 qiangli 强制封禁
- ✅ 即使数据库中设置了 qiangli=true，也不会被封禁

---

### 2. **case.js** - 案例页面

#### 2.1 环境检测（模拟器/预览模式）
**位置**: `miniprogram/pages/case/case.js` 第 226-244 行

**功能**: 检测运行环境，判断是否为模拟器或预览模式
```javascript
// 模拟器通常 platform 是 'devtools'
const isSimulator = sysInfo.platform === 'devtools' || 
                    sysInfo.system.indexOf('devtools') !== -1 ||
                    !sysInfo.brand || 
                    sysInfo.model === 'devtools';

// 检测预览模式（通过二维码扫描进入）
const isPreview = sysInfo.platform !== 'devtools' && 
                  !sysInfo.brand && 
                  !sysInfo.model;
```

**影响**:
- ✅ 模拟器环境会使用自定义选择器（`useCustomPicker: true`）
- ✅ 预览模式会标记为 `isPreview: true`

---

### 3. **blocked.js** - 封禁页面

#### 3.1 已移除开发工具限制
**位置**: `miniprogram/pages/blocked/blocked.js` 第 55-57 行

**说明**: 
- 之前代码：开发环境下不启动云函数检查，避免自动解封
- 现在：**已移除限制**，允许开发工具环境也执行检查，方便调试 Auto 功能

**影响**:
- ✅ 开发工具中也会执行 Auto 模式检查
- ✅ 可以正常调试 Auto 功能

---

## 📊 总结

### 开发工具环境下会跳过的功能：
1. ✅ **封禁状态检查**（应用启动时）
2. ✅ **PC端检测**（Windows/Mac微信封禁）
3. ✅ **qiangli 强制封禁检查**（定时检查）

### 开发工具环境下会启用的功能：
1. ✅ **环境检测**（标记为模拟器）
2. ✅ **自定义选择器**（case页面）
3. ✅ **Auto 模式检查**（blocked页面，已移除限制）

### 注意事项：
- ⚠️ 开发工具环境下的行为可能与真机不同
- ⚠️ 某些功能（如截屏/录屏检测）在开发工具中可能无法正常工作
- ⚠️ 建议在真机上测试关键功能

---

## 🔧 如何修改

如果需要修改开发工具环境的行为，可以：

1. **修改检测逻辑**：
   ```javascript
   const isDevTools = deviceInfo.platform === 'devtools';
   ```

2. **移除开发工具限制**：
   删除或注释掉 `if (isDevTools) { return; }` 相关代码

3. **添加开发工具特殊处理**：
   ```javascript
   if (isDevTools) {
     // 开发工具特殊逻辑
   } else {
     // 正常逻辑
   }
   ```
