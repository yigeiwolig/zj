# 设备数据存储和查询说明

## 设备数据存储位置

**集合名称：`sn`**

## 查询条件

### 1. 用户端查询设备列表（已激活的设备）
```javascript
db.collection('sn').where({
  openid: this.data.myOpenid,  // 当前用户的 openid
  isActive: true                // 必须是审核通过的设备
}).get()
```

### 2. 调试查询（所有设备，包括未激活的）
```javascript
db.collection('sn').where({
  openid: this.data.myOpenid
}).get()
```

## 设备数据的关键字段

- `sn`: 设备序列号（纯数字，不包含 MT 前缀）
- `openid`: 设备所属用户的 openid（必须字段）
- `isActive`: 是否已激活（true = 审核通过，false = 待审核）
- `productModel`: 产品型号（如 "F1 PRO"）
- `expiryDate`: 质保到期日期
- `activations`: 激活次数
- `firmware`: 固件版本
- `bindTime`: 绑定时间
- `imgReceipt`: 购买截图

## 数据流程

1. **用户提交审核** → 数据存入 `my_read` 集合（状态：PENDING）
2. **管理员审核通过** → 云函数 `adminAuditDevice` 更新 `sn` 集合：
   - 设置 `openid` = 申请人的 openid
   - 设置 `isActive` = true
   - 设置其他设备信息（型号、到期日等）
3. **用户端查询** → 从 `sn` 集合查询 `openid` 匹配且 `isActive = true` 的设备

## 检查数据的方法

### 在云开发控制台查询：

1. 打开云开发控制台
2. 进入数据库
3. 选择 `sn` 集合
4. 查询条件：
   - 查询所有设备：不设置条件
   - 查询特定用户的设备：`openid` = 用户的 openid
   - 查询已激活的设备：`isActive` = true
   - 查询未激活的设备：`isActive` = false 或不存在

### 检查要点：

1. **设备是否存在**：查询 `sn` 集合，看是否有对应 SN 的记录
2. **openid 是否正确**：检查设备的 `openid` 字段是否等于用户的 openid
3. **isActive 是否为 true**：检查设备的 `isActive` 字段是否为 true
4. **数据是否完整**：检查 `productModel`、`expiryDate` 等字段是否存在

## 常见问题

### 问题1：设备列表为空
- 检查 `sn` 集合中是否有该 `openid` 的记录
- 检查 `isActive` 是否为 `true`
- 检查 `openid` 字段是否正确设置

### 问题2：审核通过后设备不显示
- 检查云函数 `adminAuditDevice` 是否正确设置了 `openid`
- 检查云函数日志，确认更新是否成功
- 检查设备的 `isActive` 字段是否为 `true`

### 问题3：openid 不匹配
- 检查 `my_read` 集合中的 `_openid` 是否正确
- 检查云函数是否正确获取了 `_openid`
- 检查更新时是否正确设置了 `openid` 字段
