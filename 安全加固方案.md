# 小程序安全加固方案

## 一、当前安全风险分析

### 1. 敏感信息暴露风险
- ✅ **云函数中的支付密钥**：已存储在云函数中（相对安全）
- ⚠️ **环境ID暴露**：`cloudbase-4gn1heip7c38ec6c` 在小程序代码中可见
- ⚠️ **数据库集合名称暴露**：所有集合名称都在代码中可见

### 2. 数据访问风险
- ⚠️ **小程序端直接访问数据库**：依赖微信的 `_openid` 自动注入机制
- ⚠️ **云函数调用无额外验证**：仅依赖微信身份验证
- ⚠️ **无请求签名机制**：容易被恶意调用

### 3. 爬虫风险
- ⚠️ **无频率限制**：可能被频繁调用
- ⚠️ **无请求来源验证**：无法区分正常用户和爬虫

## 二、安全加固措施

### 1. 数据库权限加固（最重要！）

#### 步骤1：检查并设置数据库权限
1. 打开微信开发者工具 → 云开发 → 数据库
2. 对每个集合设置权限规则：

**推荐权限设置：**

```json
// 示例：shouhou_repair 集合权限规则
{
  "read": "doc._openid == auth.openid || doc.openid == auth.openid",
  "write": "doc._openid == auth.openid || doc.openid == auth.openid"
}
```

**关键集合权限建议：**
- `sn`：仅创建者可读写
- `shouhou_repair`：仅创建者可读写
- `my_read`：仅创建者可读写
- `video`：仅创建者可读写
- `shop_orders`：仅创建者可读写
- `guanliyuan`：仅管理员可读，仅云函数可写
- `login_logbutton`：仅云函数可读写
- `valid_users`：仅云函数可读写

### 2. 云函数安全加固

#### 2.1 添加请求签名验证

创建 `cloudfunctions/verifyRequest/index.js`：

```javascript
const crypto = require('crypto');

exports.main = async (event, context) => {
  const { timestamp, signature, data } = event;
  
  // 从环境变量获取密钥（在云开发控制台设置）
  const SECRET_KEY = process.env.REQUEST_SECRET_KEY || 'your-secret-key';
  
  // 验证时间戳（防止重放攻击，5分钟内有效）
  const now = Date.now();
  if (Math.abs(now - timestamp) > 5 * 60 * 1000) {
    return { success: false, error: '请求已过期' };
  }
  
  // 验证签名
  const expectedSignature = crypto
    .createHash('sha256')
    .update(`${timestamp}${JSON.stringify(data)}${SECRET_KEY}`)
    .digest('hex');
  
  if (signature !== expectedSignature) {
    return { success: false, error: '签名验证失败' };
  }
  
  return { success: true };
};
```

#### 2.2 添加频率限制

创建 `cloudfunctions/rateLimit/index.js`：

```javascript
const cloud = require('wx-server-sdk');
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV });
const db = cloud.database();

exports.main = async (event, context) => {
  const wxContext = cloud.getWXContext();
  const openid = wxContext.OPENID;
  const { action, maxCount = 10, windowMs = 60000 } = event; // 默认1分钟内最多10次
  
  const now = Date.now();
  const windowStart = now - windowMs;
  
  // 查询该用户在时间窗口内的请求次数
  const countRes = await db.collection('rate_limit_logs')
    .where({
      openid: openid,
      action: action,
      timestamp: db.command.gte(windowStart)
    })
    .count();
  
  if (countRes.total >= maxCount) {
    return { 
      success: false, 
      error: '请求过于频繁，请稍后再试',
      retryAfter: Math.ceil((windowMs - (now - windowStart)) / 1000)
    };
  }
  
  // 记录本次请求
  await db.collection('rate_limit_logs').add({
    data: {
      openid: openid,
      action: action,
      timestamp: now
    }
  });
  
  return { success: true };
};
```

#### 2.3 敏感信息使用环境变量

**在云开发控制台设置环境变量：**
1. 打开微信开发者工具 → 云开发 → 云函数
2. 选择云函数 → 配置 → 环境变量
3. 添加以下环境变量：
   - `WX_PAY_MCH_ID`：商户号
   - `WX_PAY_API_V3_KEY`：API V3密钥
   - `WX_PAY_SERIAL_NO`：证书序列号
   - `REQUEST_SECRET_KEY`：请求签名密钥

**修改云函数代码使用环境变量：**

```javascript
// 从环境变量读取
const WX_PAY_CONFIG = {
  mchId: process.env.WX_PAY_MCH_ID,
  apiV3Key: process.env.WX_PAY_API_V3_KEY,
  serialNo: process.env.WX_PAY_SERIAL_NO,
  // ...
};
```

### 3. 小程序端安全加固

#### 3.1 添加请求加密（可选，高级）

如果需要对敏感数据进行加密传输，可以使用 AES 加密：

```javascript
// utils/crypto.js
const CryptoJS = require('crypto-js');

// 加密（密钥从云函数获取，不要写死在小程序中）
function encrypt(data, key) {
  return CryptoJS.AES.encrypt(JSON.stringify(data), key).toString();
}

// 解密
function decrypt(encryptedData, key) {
  const bytes = CryptoJS.AES.decrypt(encryptedData, key);
  return JSON.parse(bytes.toString(CryptoJS.enc.Utf8));
}
```

#### 3.2 添加请求签名（小程序端）

```javascript
// utils/request.js
const crypto = require('crypto');

function signRequest(data) {
  const timestamp = Date.now();
  const SECRET_KEY = 'your-secret-key'; // 从云函数获取，不要写死
  const signature = crypto
    .createHash('sha256')
    .update(`${timestamp}${JSON.stringify(data)}${SECRET_KEY}`)
    .digest('hex');
  
  return { timestamp, signature, data };
}
```

### 4. 数据加密存储（可选）

对于特别敏感的数据，可以在存储前加密：

```javascript
// 云函数中加密存储
const crypto = require('crypto');

function encryptField(value, key) {
  const cipher = crypto.createCipher('aes-256-cbc', key);
  let encrypted = cipher.update(value, 'utf8', 'hex');
  encrypted += cipher.final('hex');
  return encrypted;
}

function decryptField(encrypted, key) {
  const decipher = crypto.createDecipher('aes-256-cbc', key);
  let decrypted = decipher.update(encrypted, 'hex', 'utf8');
  decrypted += decipher.final('utf8');
  return decrypted;
}
```

## 三、实施优先级

### 🔴 高优先级（必须立即实施）
1. **数据库权限设置**：设置所有集合的权限规则，确保用户只能访问自己的数据
2. **云函数环境变量**：将敏感信息移到环境变量
3. **频率限制**：添加请求频率限制，防止爬虫

### 🟡 中优先级（建议实施）
4. **请求签名验证**：在关键云函数中添加签名验证
5. **日志监控**：记录异常请求，及时发现攻击

### 🟢 低优先级（可选）
6. **数据加密存储**：对特别敏感的数据进行加密
7. **请求加密传输**：使用 HTTPS + 额外加密层

// 记录异常请求

### 2. 定期检查

- 每周检查数据库访问日志
- 监控异常请求频率

## 六、注意事项

1. **不要在小程序代码中硬编码密钥**
2. **数据库权限是最后一道防线，必须设置**
3. **云函数环境变量比代码中的常量更安全**
4. **频率限制可以有效防止爬虫**
5. **定期更新密钥和证书**
