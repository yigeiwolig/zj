# 数据库权限设置建议

## ✅ 可以设置为"仅创建者可读写"的集合

这些集合是用户自己创建的个人数据，不需要管理员审核，可以安全地设置为"仅创建者可读写"：

- **说明**: 用户只能访问自己的记录

### 2. **ota_connections** - OTA 连接记录

- **说明**: 用户只能访问自己的连接记录

### 3. **pending_warranty** - 待生效延保记录
- **用途**: 存储用户自己的待生效延保记录
- **权限**: 仅创建者可读写
- **说明**: 用户只能访问自己的延保记录

### 4. **shouhouguoqi** - 售后过期记录
- **用途**: 存储用户自己的售后过期记录
- **权限**: 仅创建者可读写
- **说明**: 用户只能访问自己的过期记录

### 5. **shouhou_read** - 售后阅读记录

## ⚠️ 不能设置为"仅创建者可读写"的集合

这些集合需要管理员审核或处理，必须使用自定义权限规则：

### 1. **sn** - 设备集合
- **用途**: 存储设备绑定信息
- **权限规则**:
  ```json
  {
    "read": "doc.openid == auth.openid || doc._openid == auth.openid",
    "write": "doc.openid == auth.openid || doc._openid == auth.openid"
  }
  ```
- **说明**: 用户只能读写自己的设备，但管理员需要通过云函数查询所有设备进行审核

### 2. **my_read** - 设备绑定申请

- **说明**: 用户只能读写自己的申请，但管理员需要通过云函数查询所有申请进行审核

## 🔒 需要特殊权限的集合

### 1. **guanliyuan** - 管理员集合
- **用途**: 存储管理员 openid
- **权限规则**:
  ```json
  {
    "read": "auth.openid in get('/database/guanliyuan').data.map(d => d.openid || d._openid)",
    "write": false
  }
  ```
- **说明**: 只有管理员可以读，只有云函数可以写

### 2. **valid_users** - 白名单

## 📝 设置步骤

1. 打开微信开发者工具 → 云开发 → 数据库
2. 选择要设置的集合
3. 点击"权限设置"
4. 选择对应的权限规则：
   - **仅创建者可读写**: 适用于个人数据集合
   - **自定义规则**: 适用于需要管理员审核的集合
   - **仅云函数可读写**: 适用于系统数据集合

---

## ⚠️ 注意事项

1. **管理员审核功能**: 需要管理员审核的集合（如 `sn`、`my_read`、`shouhou_repair`），管理员需要通过云函数来查询所有记录，因为小程序端权限限制了只能查询自己的数据。

2. **兼容性**: 某些集合可能同时使用 `openid` 和 `_openid` 字段，权限规则需要兼容两种情况。

3. **测试**: 修改权限后，务必测试用户端和管理员端的功能是否正常。
